# Session 12 — Text Alignment

در این جلسه تراز کردن متن داخل یک باکس را می‌بینیم. عنصر `.box` یک ظرف با عرض ثابت است و متن‌های داخلش با خاصیت‌های مختلف تراز می‌شوند.

```css
.box {
  width: 600px;
  padding: 20px;
  /* ... */
}
```

---

## 1. `text-align`
جهت افقی متن (و محتوای inline) را داخل عنصر مشخص می‌کند.

| مقدار | معنی |
| --- | --- |
| `left` | چپ‌چین (پیش‌فرض در زبان‌های LTR مثل انگلیسی) |
| `right` | راست‌چین |
| `center` | وسط‌چین |
| `justify` | تراز دو طرفه؛ فاصله بین کلمات طوری تنظیم می‌شود که هر دو لبه پر شوند |

### مثال: وسط‌چین کردن عنوان

```css
.box h1 {
  text-align: center;
}
```

### مثال: justify برای پاراگراف

```css
.box p {
  text-align: justify;
}
```

با `justify`، خطوط میانی پاراگراف از چپ و راست هم‌تراز می‌شوند. خط آخر معمولاً مثل حالت عادی می‌ماند (چپ‌چین در LTR).

---

## 2. `text-align-last`
فقط **آخرین خط** پاراگراف را جداگانه تراز می‌کند.

```css
.box p {
  text-align: justify;
  text-align-last: center;
}
```

یعنی:
- خطوط معمولی → `justify`
- خط آخر → `center`

مقادیر رایج مثل `left`، `right`، `center` و `justify` را می‌پذیرد.

---

## 3. `direction`
جهت نوشتار را مشخص می‌کند.

| مقدار | معنی |
| --- | --- |
| `ltr` | چپ به راست (Left to Right) — مثل انگلیسی |
| `rtl` | راست به چپ (Right to Left) — مثل فارسی/عربی |

```css
.box .input {
  direction: ltr;
}
```

`direction` روی ترتیب متن و حتی معنای `text-align` پیش‌فرض اثر دارد. در صفحات فارسی معمولاً روی `body` یا بخش‌های فارسی `direction: rtl` می‌گذارند.

---

## خلاصه مثال‌های این جلسه

| عنصر | خاصیت | نتیجه |
| --- | --- | --- |
| `h1` | `text-align: center` | عنوان وسط باکس |
| `p` | `text-align: justify` | پاراگراف دو طرفه |
| `p` | `text-align-last: center` | خط آخر وسط‌چین |
| `input` | `direction: ltr` | جهت ورودی چپ به راست |

> نکته: `text-align` روی محتوای inline داخل بلاک اثر دارد؛ برای وسط‌چین کردن خود یک باکس بلوکی معمولاً از `margin: auto` یا Flex/Grid استفاده می‌شود.
