# Session 19 — Position

`position` مشخص می‌کند عنصر نسبت به چه چیزی جابه‌جا شود. بعد از تعیین `position`، با `top`، `right`، `bottom`، `left` جایش را تنظیم می‌کنیم.

پیش‌فرض همه عناصر `position: static` است؛ در آن حالت `top`/`left` و ... اثر ندارند.

در این جلسه چهار حالت را می‌بینیم: **`relative`**، **`fixed`**، **`absolute`**، **`sticky`**.

برای تست اسکرول، فضای بلند داریم:

```css
.scroll {
  height: 1000px;
}
```

---

## 1. `position: relative`
عنصر نسبت به **جای عادی خودش** جابه‌جا می‌شود.

جایش در جریان صفحه حفظ می‌شود؛ فقط ظاهرش جابه‌جا می‌شود.

```css
.box1 {
  position: relative;
  top: 300px;
}

.box2 {
  position: relative;
  bottom: 110px;
}
```

| خاصیت | اثر |
| --- | --- |
| `top` | به پایین می‌رود |
| `bottom` | به بالا می‌رود |
| `left` | به راست می‌رود |
| `right` | به چپ می‌رود |

`relative` یک نقش مهم دیگر هم دارد: برای فرزند `absolute` به‌عنوان **مرجع موقعیت** عمل می‌کند.

---

## 2. `position: fixed`
عنصر نسبت به **پنجره مرورگر (viewport)** ثابت می‌ماند و با اسکرول حرکت نمی‌کند.

```css
.box3 {
  position: fixed;
  bottom: 0;
  width: 100%;
}
```

FOOTER به پایین صفحه می‌چسبد. کاربرد رایج: هدر چسبان، فوتر ثابت، دکمه شناور.

---

## 3. `position: absolute`
عنصر از جریان عادی خارج می‌شود و نسبت به **نزدیک‌ترین والدِ غیر-static** موقعیت می‌گیرد.

اگر والدی با `relative` / `absolute` / `fixed` / `sticky` نباشد، نسبت به کل صفحه (`html`) تنظیم می‌شود.

در این مثال، `.box1` با `relative` والد است و `.box1-item` داخلش absolute شده:

```css
.box1 {
  position: relative; /* مرجع برای فرزند absolute */
}

.box1-item {
  position: absolute;
  top: 0;
  left: 0;
  width: 30%;
  height: 50%;
}
```

یعنی باکس سفید HEADER نسبت به گوشه بالا-چپِ `.box1` قرار می‌گیرد، نه نسبت به کل صفحه.

تفاوت با `fixed`:
- `absolute` → نسبت به والدِ موقعیت‌دار
- `fixed` → نسبت به viewport

---

## 4. `position: sticky`
ترکیبی از حالت عادی و ثابت است:

1. اول مثل عنصر عادی در جریان صفحه است
2. وقتی با اسکرول به آستانه برسد (مثلاً `top: 0`)، مثل `fixed` می‌چسبد
3. تا وقتی داخل ظرف والدش جا دارد می‌ماند

```css
.stickybox {
  position: sticky;
  top: 0;
  width: 100%;
  height: 100px;
}
```

صفحه را اسکرول کنید: باکس STICKY وقتی به بالای viewport برسد می‌چسبد.

> برای کار کردن sticky معمولاً باید `top` (یا یکی از جهت‌ها) تعریف شده باشد و والد overflow مخفیِ مزاحم نداشته باشد.

---

## مقایسه سریع

| موضوع | `relative` | `absolute` | `fixed` | `sticky` |
| --- | --- | --- | --- | --- |
| نسبت به چه؟ | جای خودش | والدِ موقعیت‌دار | viewport | جریان + آستانه اسکرول |
| جا در layout؟ | حفظ می‌شود | خارج می‌شود | خارج می‌شود | تا قبل چسبیدن حفظ می‌شود |
| با اسکرول؟ | همراه محتوا | همراه والد/محتوا | ثابت می‌ماند | بعد از آستانه می‌چسبد |
| مثال | HEADER / MAIN | `.box1-item` | FOOTER | `.stickybox` |

---

## خلاصه مثال‌ها

| کلاس | `position` | تنظیمات | نتیجه |
| --- | --- | --- | --- |
| `.box1` | `relative` | `top: 300px` | HEADER جابه‌جا + مرجع absolute |
| `.box2` | `relative` | `bottom: 110px` | MAIN بالاتر |
| `.box3` | `fixed` | `bottom: 0` | FOOTER ثابت پایین صفحه |
| `.box1-item` | `absolute` | `top: 0; left: 0` | داخل HEADER، گوشه بالا-چپ |
| `.stickybox` | `sticky` | `top: 0` | با اسکرول به بالا می‌چسبد |

اسکرول کنید: FOOTER همیشه پایین است، STICKY بعد از رسیدن به بالا می‌چسبد، و HEADER/MAIN با محتوا حرکت می‌کنند.
