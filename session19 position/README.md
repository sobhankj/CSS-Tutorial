# Session 19 — Position (`relative` & `fixed`)

`position` مشخص می‌کند عنصر نسبت به چه چیزی جابه‌جا شود. بعد از تعیین `position`، با `top`، `right`، `bottom`، `left` جایش را تنظیم می‌کنیم.

پیش‌فرض همه عناصر `position: static` است؛ در آن حالت `top`/`left` و ... اثر ندارند.

در این جلسه فعلاً دو حالت را می‌بینیم: **`relative`** و **`fixed`**.

---

## آماده‌سازی صفحه

سه باکس HEADER / MAIN / FOOTER داریم، به‌علاوه یک فضای بلند برای اسکرول:

```css
.scroll {
  height: 1000px;
}
```

با اسکرول می‌توانید تفاوت `relative` و `fixed` را بهتر ببینید.

---

## 1. `position: relative`
عنصر نسبت به **جای عادی خودش** جابه‌جا می‌شود.

نکته مهم: جایش در جریان صفحه (جای اصلی) حفظ می‌شود؛ فقط ظاهرش جابه‌جا می‌شود و بقیه عناصر معمولاً مثل قبل سر جایشان می‌مانند.

### مثال HEADER

```css
.box1 {
  position: relative;
  top: 300px; /* 300px به پایین نسبت به جای خودش */
}
```

### مثال MAIN

```css
.box2 {
  position: relative;
  bottom: 110px; /* 110px به بالا نسبت به جای خودش */
}
```

| خاصیت | اثر در `relative` |
| --- | --- |
| `top` | از بالا فاصله می‌گیرد (به پایین می‌رود) |
| `bottom` | از پایین فاصله می‌گیرد (به بالا می‌رود) |
| `left` | از چپ فاصله می‌گیرد (به راست می‌رود) |
| `right` | از راست فاصله می‌گیرد (به چپ می‌رود) |

---

## 2. `position: fixed`
عنصر نسبت به **پنجره مرورگر (viewport)** ثابت می‌ماند.

با اسکرول صفحه، جایش عوض نمی‌شود. برای هدر چسبان، فوتر ثابت، یا دکمه «برگشت به بالا» رایج است.

```css
.box3 {
  position: fixed;
  bottom: 0;
  width: 100%;
}
```

یعنی FOOTER به پایین صفحه می‌چسبد و هنگام اسکرول همان‌جا می‌ماند.

> چون از جریان عادی خارج می‌شود، ممکن است روی محتوای دیگر بیفتد؛ برای همین معمولاً `z-index` هم تنظیم می‌کنند (در جلسات بعد).

---

## مقایسه سریع

| موضوع | `relative` | `fixed` |
| --- | --- | --- |
| نسبت به چه؟ | جای عادی خودش | پنجره مرورگر |
| با اسکرول؟ | همراه محتوا حرکت می‌کند | سر جایش می‌ماند |
| جا در layout؟ | جای اصلی‌اش حفظ می‌شود | از جریان خارج می‌شود |
| مثال این جلسه | HEADER و MAIN | FOOTER پایین صفحه |

---

## خلاصه مثال‌ها

| کلاس | `position` | جابه‌جایی | نتیجه |
| --- | --- | --- | --- |
| `.box1` | `relative` | `top: 300px` | HEADER پایین‌تر از جایش |
| `.box2` | `relative` | `bottom: 110px` | MAIN بالاتر از جایش |
| `.box3` | `fixed` | `bottom: 0` | FOOTER ثابت در پایین viewport |

صفحه را اسکرول کنید: FOOTER ثابت می‌ماند، ولی HEADER و MAIN با محتوا حرکت می‌کنند.

> مقادیر دیگر مثل `absolute` و `sticky` در این جلسه فعلاً پوشش داده نشده‌اند.
