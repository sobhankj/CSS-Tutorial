# Session 9 — Width & Height (واحدهای اندازه‌گیری)

در این جلسه `width` و `height` را با واحدهای مختلف می‌نویسیم. انتخاب واحد درست باعث می‌شود صفحه در اندازه‌های مختلف بهتر رفتار کند.

---

## 1. `px` (پیکسل)
واحد **ثابت** است؛ مقدارش معمولاً عوض نمی‌شود.

```css
.boxParent {
  width: 600px;
  height: 600px;
}

.boxChild {
  width: 300px;
  height: 300px;
}
```

مناسب وقتی اندازه دقیق و ثابت می‌خواهید. نقطه ضعف: با زوم فونت یا تغییر اندازه صفحه، خودش مقیاس‌پذیر نیست.

---

## 2. `%` (درصد)
نسبت به **عنصر والد** محاسبه می‌شود.

```css
.boxChildInner {
  width: 50%;  /* نصف عرض والد (.boxChild) */
  height: 70%; /* ۷۰٪ ارتفاع والد */
}
```

اگر والد `300px` باشد:
- `width: 50%` → `150px`
- `height: 70%` → `210px`

اگر اندازه والد عوض شود، فرزند درصدی هم عوض می‌شود.

---

## 3. `rem` (Root EM)
نسبت به **`font-size` ریشه صفحه** (`html`) محاسبه می‌شود. مقدار پیش‌فرض معمولاً `16px` است.

```css
.boxChildInnerInner {
  width: 5.5rem;  /* 5.5 × 16 = 88px */
  height: 3.5rem; /* 3.5 × 16 = 56px */
}
```

فرمول:

```text
مقدار نهایی = عدد rem × font-size ریشه
```

مزیت مهم: اگر کاربر اندازه فونت مرورگر را بزرگ کند، عناصر `rem` هم بزرگ می‌شوند → دسترس‌پذیری بهتر.

---

## 4. `em`
نسبت به **`font-size` والد مستقیم** (یا خود عنصر، بسته به خاصیت) محاسبه می‌شود.

```css
h1 {
  font-size: 3rem; /* نسبت به ریشه */
}

span {
  font-size: 0.5em; /* نصف font-size والد خودش */
}
```

اگر والد `32px` باشد، `0.5em` می‌شود `16px`.

تفاوت کلیدی با `rem`:

| واحد | نسبت به چیست؟ |
| --- | --- |
| `rem` | همیشه ریشه (`html`) |
| `em` | والد (قابل تغییر در هر لایه) |

`em` می‌تواند تو در تو بزرگ یا کوچک شود؛ برای همین گاهی گیج‌کننده است. برای اندازه‌های کلی معمولاً `rem` ساده‌تر است.

---

## 5. `vw` (Viewport Width)
نسبت به **عرض پنجره مرورگر**.

```text
1vw = 1٪ عرض viewport
```

```css
.boxParent2 {
  width: 70vw; /* ۷۰٪ عرض صفحه */
}
```

اگر عرض صفحه `1000px` باشد، `70vw` ≈ `700px`. با تغییر اندازه پنجره، عرض باکس هم عوض می‌شود.

---

## 6. `vh` (Viewport Height)
نسبت به **ارتفاع پنجره مرورگر**.

```text
1vh = 1٪ ارتفاع viewport
```

```css
.boxParent2 {
  height: 50vh; /* نصف ارتفاع صفحه */
}
```

اگر ارتفاع صفحه `800px` باشد، `50vh` = `400px`.

---

## مقایسه سریع

| واحد | نسبت به | ثابت / نسبی | کاربرد رایج |
| --- | --- | --- | --- |
| `px` | پیکسل صفحه | ثابت | اندازه‌های دقیق |
| `%` | والد | نسبی | عرض/ارتفاع داخل یک باکس |
| `rem` | `html` | نسبی | تایپوگرافی و فاصله‌های مقیاس‌پذیر |
| `em` | والد | نسبی | اندازه فونت وابسته به والد |
| `vw` | عرض viewport | نسبی | بخش‌هایی که با عرض صفحه تغییر کنند |
| `vh` | ارتفاع viewport | نسبی | hero، تمام‌صفحه، ارتفاع نسبی به پنجره |

---

## خلاصه ساختار مثال این جلسه

```text
.boxParent        → 600px × 600px   (سبز)
  .boxChild       → 300px × 300px   (سیاه)
    .boxChildInner → 50% × 70%      (قرمز)
      .boxChildInnerInner → 5.5rem × 3.5rem  (بنفش)

.boxParent2       → 70vw × 50vh     (صورتی)
```

با تغییر اندازه پنجره، اثر `vw` و `vh` را روی `.boxParent2` ببینید؛ `%` فقط وقتی والد تغییر کند عوض می‌شود.
