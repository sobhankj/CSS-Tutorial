# Session 29 — Transform

`transform` شکل، اندازه یا موقعیت یک عنصر را **روی صفحه** تغییر می‌دهد؛ بدون اینکه معمولاً روی layout بقیه‌ی عناصر اثر بگذارد (برخلاف `margin` یا تغییر `width`).

در این جلسه پنج نوع مهم را با hover می‌بینیم:

- `rotate` — چرخش
- `scale` — بزرگ/کوچک کردن
- `translate` — جابه‌جایی
- `skew` — کج کردن
- `matrix` — ترکیب پیشرفته با ماتریس

همه با `transition: all 0.5s ease 20ms` نرم انیمیت می‌شوند.

---

## 1. شکل کلی

```css
transform: function(value);
```

چند تابع را می‌توان پشت سر هم نوشت:

```css
transform: rotate(45deg) scale(1.2);
```

ترتیب مهم است؛ اولی اول اعمال می‌شود.

---

## 2. `rotate` — چرخش

```css
.box1:hover {
  transform: rotate(45deg);
}
```

| بخش | معنی |
| --- | --- |
| `rotate` | چرخش حول مرکز عنصر |
| `45deg` | زاویه چرخش (درجه) |

مربع آبی با hover ۴۵ درجه می‌چرخد.

مقادیر رایج:

```css
rotate(90deg);   /* یک‌چهارم دایره */
rotate(-45deg);  /* خلاف جهت عقربه‌ها */
rotate(1turn);   /* یک دور کامل */
```

---

## 3. `scale` — تغییر اندازه

```css
.box2:hover {
  transform: scale(1.5);
}
```

| بخش | معنی |
| --- | --- |
| `scale(1.5)` | ۱.۵ برابر در هر دو محور |
| `1` | اندازه اصلی |
| کمتر از `1` | کوچک‌تر |

مربع قرمز با hover ۱.۵ برابر بزرگ می‌شود.

محور جداگانه:

```css
scale(2);        /* هر دو محور */
scaleX(1.5);     /* فقط عرض */
scaleY(0.5);     /* فقط ارتفاع */
scale(2, 0.5);   /* X و Y جدا */
```

---

## 4. `translate` — جابه‌جایی

```css
.box3:hover {
  transform: translate(100px, 100px);
}
```

| بخش | معنی |
| --- | --- |
| `100px` اول | جابه‌جایی افقی (مثبت = راست) |
| `100px` دوم | جابه‌جایی عمودی (مثبت = پایین) |

مربع سبز با hover ۱۰۰px به راست و ۱۰۰px پایین می‌رود.

شکل‌های رایج:

```css
translate(50px);       /* فقط افقی */
translateX(50px);
translateY(-20px);     /* منفی = بالا */
translate(10%, 20%);
```

نکته: `translate` معمولاً برای حرکت روان‌تر از تغییر `top`/`left` است.

---

## 5. `skew` — کج کردن

```css
.box4:hover {
  transform: skew(20deg, 10deg);
}
```

| بخش | معنی |
| --- | --- |
| `20deg` | کج شدن در محور X |
| `10deg` | کج شدن در محور Y |

مربع فیروزه‌ای با hover به شکل متوازی‌الاضلاع کج می‌شود.

```css
skewX(20deg);
skewY(10deg);
skew(20deg);      /* فقط X */
```

---

## 6. `matrix` — فرم ماتریسی

```css
.box5:hover {
  transform: matrix(1, 0, 0, 1, 100, 100);
}
```

شکل کلی:

```css
matrix(a, b, c, d, tx, ty);
```

| پارامتر | نقش تقریبی |
| --- | --- |
| `a`, `d` | مقیاس (scale) در X و Y |
| `b`, `c` | کج شدن / چرخش |
| `tx`, `ty` | جابه‌جایی (مثل translate) |

در این مثال `matrix(1, 0, 0, 1, 100, 100)` معادل تقریبی `translate(100px, 100px)` است؛ مربع بنفش فقط جابه‌جا می‌شود.

`matrix` قدرتمند است ولی خوانایی کمتری دارد؛ معمولاً از `rotate` / `scale` / `translate` / `skew` استفاده می‌کنیم مگر نیاز خاصی باشد.

---

## 7. نکته مهم درباره layout

`transform` معمولاً **جای عنصر در flow را عوض نمی‌کند**. یعنی عنصرهای دیگر جایشان ثابت می‌ماند؛ فقط ظاهر عنصر جابه‌جا یا تغییر شکل می‌دهد.

این با `margin` یا تغییر `width`/`height` فرق دارد.

---

## 8. ترکیب با `transition`

در این جلسه همه باکس‌ها این را دارند:

```css
transition: all 0.5s ease 20ms;
```

بدون transition، `transform` ناگهان اعمال می‌شود. با transition، چرخش، بزرگ شدن و بقیه حرکت‌ها نرم دیده می‌شوند.

---

## خلاصه مثال‌های این جلسه

| کلاس | رنگ | `transform` روی hover | نتیجه |
| --- | --- | --- | --- |
| `.box1` | آبی | `rotate(45deg)` | چرخش ۴۵ درجه |
| `.box2` | قرمز | `scale(1.5)` | ۱.۵ برابر بزرگ‌تر |
| `.box3` | سبز | `translate(100px, 100px)` | جابه‌جایی راست و پایین |
| `.box4` | فیروزه‌ای | `skew(20deg, 10deg)` | کج شدن |
| `.box5` | بنفش | `matrix(1, 0, 0, 1, 100, 100)` | جابه‌جایی با ماتریس |

پنج مربع زیر هم در `.container`؛ با hover روی هر کدام، transform مربوطه با انیمیشن نیم‌ثانیه‌ای اجرا می‌شود.
