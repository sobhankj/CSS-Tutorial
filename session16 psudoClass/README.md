# Session 16 — Pseudo-classes

Pseudo-class با `:` نوشته می‌شود و وقتی عنصر در یک **حالت خاص** باشد، استایل اعمال می‌کند (مثلاً هاور، فوکوس، کلیک).

شکل کلی:

```css
selector:pseudo-class {
  /* styles */
}
```

در این جلسه چهار مورد مهم را می‌بینیم: `:hover`، `:active`، `:visited`، `:focus`

---

## 1. `:hover`
وقتی ماوس روی عنصر باشد.

```css
.btn:hover {
  background-color: azure;
  border: 1px solid #ff0000;
}
```

روی دکمه بروید: پس‌زمینه روشن‌تر و border قرمز می‌شود.

---

## 2. `:active`
لحظه‌ای که عنصر را **کلیک/فشار** می‌دهید (تا وقتی دکمه ماوس پایین است).

```css
.btn:active {
  background-color: #000;
  color: #fff;
  border: 1px solid #00ff48;
}
```

دکمه را نگه دارید: پس‌زمینه سیاه و متن سفید می‌شود. بعد از رها کردن، به حالت عادی (یا hover) برمی‌گردد.

---

## 3. `:visited`
برای لینک‌هایی که قبلاً باز شده‌اند.

```css
a {
  text-decoration: none;
  color: #000;
}

a:visited {
  color: #00ff48;
}
```

بعد از یک بار کلیک روی لینک Google و برگشت به صفحه، رنگ لینک سبز می‌شود.

> محدودیت امنیتی مرورگر: روی `:visited` فقط چند خاصیت محدود مثل `color` قابل اعتماد هستند؛ خیلی از خاصیت‌ها (مثل `background`) ممکن است اعمال نشوند.

---

## 4. `:focus`
وقتی عنصر فوکوس دارد؛ مثلاً با کلیک داخل `input` یا رفتن با کلید Tab.

```css
input:focus {
  width: 500px;
  padding: 10px;
}
```

روی input کلیک کنید: عرض و padding بیشتر می‌شود تا ورودی برجسته‌تر دیده شود.

---

## ترتیب رایج برای لینک‌ها (LVHA)
اگر چند حالت را با هم بنویسید، ترتیب پیشنهادی این است:

```text
:link → :visited → :hover → :active
```

چون اگر ترتیب اشتباه باشد، بعضی حالت‌ها ممکن است همدیگر را override کنند.

---

## خلاصه مثال‌های این جلسه

| عنصر | Pseudo-class | چه زمانی؟ | اثر |
| --- | --- | --- | --- |
| `.btn` | `:hover` | ماوس روی دکمه | پس‌زمینه azure، border قرمز |
| `.btn` | `:active` | لحظه کلیک | پس‌زمینه سیاه، متن سفید |
| `a` | `:visited` | لینک قبلاً دیده شده | رنگ سبز |
| `input` | `:focus` | فوکوس روی ورودی | عرض و padding بیشتر |

برای تست سریع: روی دکمه hover/کلیک کنید، لینک را یک بار باز کنید، و داخل input کلیک کنید.
