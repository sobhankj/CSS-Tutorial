# Session 20 — z-index

`z-index` ترتیب لایه‌بندی عناصر روی هم را مشخص می‌کند؛ مثل محور Z که عمق صفحه را نشان می‌دهد.

عدد **بزرگ‌تر** = لایه بالاتر (روی بقیه دیده می‌شود).

---

## شرط مهم
`z-index` فقط روی عناصری کار می‌کند که `position` آن‌ها غیر از `static` باشد؛ مثلاً:

- `relative`
- `absolute`
- `fixed`
- `sticky`

اگر `position` ننویسید، `z-index` معمولاً بی‌اثر است.

---

## مثال این جلسه

یک والد `relative` داریم و دو فرزند `absolute` که روی هم می‌افتند:

```css
.parent {
  position: relative;
}

.child1 {
  position: absolute;
  top: 0;
  left: 30%;
  z-index: 2; /* لایه بالاتر */
}

.child2 {
  position: absolute;
  top: 0;
  right: 40%;
  /* z-index ندارد → مثل auto / 0 رفتار می‌کند */
}
```

چون `child1` مقدار `z-index: 2` دارد و `child2` ندارد، در محل تداخل **Child1 روی Child2** دیده می‌شود.

اگر برای `child2` مثلاً `z-index: 5` بگذارید، آن وقت Child2 بالاتر می‌آید.

---

## مقادیر رایج

| مقدار | معنی |
| --- | --- |
| `auto` | پیش‌فرض؛ ترتیب مثل ترتیب HTML |
| عدد مثبت (`1`, `2`, `10`...) | لایه بالاتر |
| `0` | لایه پایه |
| عدد منفی (`-1`) | لایه پایین‌تر (ممکن است زیر والد برود) |

نیازی نیست اعداد خیلی بزرگ بگذارید؛ معمولاً `1`، `2`، `10` کافی است.

---

## Stacking Context (مفهوم کوتاه)
گاهی یک عنصر با `z-index`، لایه جداگانه‌ای برای فرزندانش می‌سازد. در آن صورت `z-index` فرزندان فقط داخل همان والد مقایسه می‌شوند، نه با کل صفحه.

در این مثال، `.parent` با `position: relative` مرجع موقعیت است و دو فرزند داخل آن با هم مقایسه می‌شوند.

---

## خلاصه

| کلاس | `position` | `z-index` | نتیجه |
| --- | --- | --- | --- |
| `.parent` | `relative` | — | ظرف و مرجع absolute |
| `.child1` | `absolute` | `2` | بالاتر دیده می‌شود |
| `.child2` | `absolute` | ندارد (`auto`) | پایین‌تر از child1 |

> یادآوری از جلسه Position: `absolute` نسبت به نزدیک‌ترین والدِ موقعیت‌دار (`relative`) تنظیم می‌شود؛ `z-index` فقط می‌گوید کدام‌شان روی دیگری باشد.
