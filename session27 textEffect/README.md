# Session 27 — Text Effect

در این جلسه با **برش متن** و نمایش **سه‌نقطه** (`...`) وقتی متن از عرض عنصر بیشتر می‌شود آشنا می‌شویم.

این الگو در UIهای واقعی زیاد دیده می‌شود: عنوان کارت، نام کاربر، توضیح کوتاه در لیست و غیره.

---

## 1. مشکل: متن طولانی در فضای محدود

اگر یک `div` با عرض ثابت داشته باشیم و متن داخلش بلند باشد، به‌صورت پیش‌فرض متن **شکسته** و به خط بعد می‌رود. گاهی می‌خواهیم متن در **یک خط** بماند و ادامه‌اش با `...` نشان داده شود.

برای این کار باید **سه property** با هم استفاده شوند.

---

## 2. سه‌گانه ضروری برای `ellipsis`

```css
.text {
  width: 300px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

| Property | نقش |
| --- | --- |
| `width: 300px` | عرض ثابت؛ بدون محدودیت عرض، ellipsis کار نمی‌کند |
| `white-space: nowrap` | متن در یک خط می‌ماند و به خط بعد نمی‌رود |
| `overflow: hidden` | بخش اضافه‌ی متن پنهان می‌شود |
| `text-overflow: ellipsis` | به‌جای برش ناگهانی، `...` نمایش داده می‌شود |

هر سه مورد `nowrap`، `hidden` و `ellipsis` با هم لازم‌اند؛ حذف هر کدام باعث می‌شود سه‌نقطه دیده نشود.

---

## 3. مثال HTML این جلسه

```html
<div class="text" title="lorem">
  Lorem ipsum dolor sit amet consectetur ...
</div>
```

متن داخل `div` بسیار بلند است، ولی فقط بخشی از آن در عرض ۳۰۰ پیکسل دیده می‌شود و انتهای خط با `...` تمام می‌شود.

### `title` برای نمایش متن کامل

ویژگی `title="lorem"` یک **tooltip** ساده در مرورگر ایجاد می‌کند؛ با hover روی عنصر، متن tooltip نمایش داده می‌شود. در پروژه‌های واقعی معمولاً `title` را با همان متن کامل یا خلاصه‌ی مفید پر می‌کنند تا کاربر بتواند متن بریده‌شده را ببیند.

---

## 4. `text-overflow` — مقادیر رایج

```css
text-overflow: clip;      /* پیش‌فرض؛ برش ساده بدون ... */
text-overflow: ellipsis;  /* نمایش سه‌نقطه */
```

| مقدار | نتیجه |
| --- | --- |
| `clip` | متن ناگهان قطع می‌شود |
| `ellipsis` | انتهای متن با `...` نشان داده می‌شود |

---

## 5. `white-space` — کنترل شکست خط

```css
white-space: normal;   /* پیش‌فرض؛ شکست خط عادی */
white-space: nowrap;   /* هیچ شکست خطی؛ یک خط */
white-space: pre;      /* فاصله‌ها و enter حفظ می‌شوند */
```

برای ellipsis تک‌خطی، `nowrap` ضروری است.

---

## 6. محدودیت تک‌خطی

این روش فقط برای **یک خط** متن کار می‌کند. برای برش چندخطی (مثلاً حداکثر ۲ یا ۳ خط) باید از روش‌های دیگر مثل `-webkit-line-clamp` استفاده کرد — موضوعی برای جلسات بعدی.

---

## خلاصه مثال این جلسه

| بخش | مقدار | نتیجه |
| --- | --- | --- |
| `.text` | `width: 300px` | محدودیت عرض |
| `.text` | `white-space: nowrap` | متن در یک خط |
| `.text` | `overflow: hidden` | مخفی کردن اضافه |
| `.text` | `text-overflow: ellipsis` | نمایش `...` |
| `title` | روی `div` | tooltip ساده هنگام hover |

یک بلوک متن بلند در عرض ۳۰۰ پیکسل دیده می‌شود؛ انتهای آن با سه‌نقطه قطع شده و با hover روی `title` می‌توان tooltip را دید.
