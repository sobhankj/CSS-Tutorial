# Session 21 — Overflow

`overflow` مشخص می‌کند وقتی محتوای داخل باکس از اندازه‌اش **بیرون بزند**، چه اتفاقی بیفتد.

در این جلسه همه باکس‌ها اندازه ثابت دارند:

```css
.container {
  height: 200px;
  width: 300px;
}
```

متن داخل‌شان طولانی است؛ پس overflow لازم می‌شود.

---

## مقادیر مهم

### 1. `overflow: hidden`
محتوای اضافه **بریده** می‌شود و دیده نمی‌شود. اسکرول هم نیست.

```css
.box1 {
  overflow: hidden;
}
```

### 2. `overflow: auto`
اگر محتوا جا نشود، در صورت نیاز **اسکرول** نشان داده می‌شود؛ اگر جا شود، اسکرول ظاهر نمی‌شود.

```css
.box2 {
  overflow: auto;
}
```

### 3. `overflow: scroll`
همیشه جای اسکرول در نظر گرفته می‌شود (حتی اگر لازم نباشد). معمولاً نوار اسکرول دیده می‌شود.

```css
.box3 {
  overflow: scroll;
}
```

### 4. `overflow: visible` (پیش‌فرض)
محتوای اضافه از باکس **بیرون می‌زند** و روی فضای اطراف دیده می‌شود.

```css
.box4 {
  overflow: visible;
}
```

---

## مقایسه سریع

| مقدار | محتوای اضافه | اسکرول |
| --- | --- | --- |
| `hidden` | بریده می‌شود | خیر |
| `auto` | داخل باکس می‌ماند | فقط وقتی لازم باشد |
| `scroll` | داخل باکس می‌ماند | معمولاً همیشه |
| `visible` | بیرون می‌زند | خیر |

---

## کنترل جداگانه محورها
می‌توان فقط افقی یا عمودی را تنظیم کرد:

```css
overflow-x: hidden;  /* فقط افقی */
overflow-y: auto;    /* فقط عمودی */
```

---

## خلاصه مثال‌های این جلسه

| کلاس | `overflow` | نتیجه |
| --- | --- | --- |
| `.box1` | `hidden` | متن اضافی مخفی |
| `.box2` | `auto` | اسکرول در صورت نیاز |
| `.box3` | `scroll` | اسکرول دائمی |
| `.box4` | `visible` | متن از باکس بیرون می‌زند |

چهار باکس را کنار هم ببینید تا تفاوت بریدن، اسکرول و بیرون زدن محتوا مشخص شود.

> نکته: `overflow` غیر از `visible` می‌تواند روی رفتارهایی مثل `position: sticky` هم اثر بگذارد (از جلسه Position).
